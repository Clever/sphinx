Sphinx API Rate Limiter

By Clever

Proceed for answers

Written in Go with redis and memory backed leakybuckets

Sphinx is an API rate limiting proxy built using leaky buckets
that can be configured with multiple limits based on request parameters.
Response headers are added for matching requests with details about the rate limit status.
It is released under the MIT License.

Features

Define Multiple Limits

Configure named limits that apply to requests based on request parameters such as headers request source.

X-RateLimit-*
headers with limit status

Sphinx automatically adds response headers for matched request that provide clients details such as limit name, remaining requests and reset time.
